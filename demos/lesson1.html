1.2: arguments and strings

alert() doesn’t do much unless we pass an argument into it.

Arguments go inside the parentheses, a.k.a. (_), of a function.

Strings are a basic type of data that represents text.
In JavaScript, we indicate strings via single or double quotation marks, a.k.a. '_' and "_".

So in this example, the argument is the string "I think you’re cool!".



In our code, we have to be careful about if we use single or double quotes.
I’m using single quotes from the string argument because double quotes would cancel out the onlick="" attribute.

1.3: numbers and math

Besides strings, there are a few types of data in JavaScript.
Another basic type of data is a number.

Numbers are cool because they can do MATH!

In JavaScript, we have the following arithmetic/math operators:

  • + for adding
  • - for subtracting
  • * for multiplying
  • ** for exponentiation
  • / for division
  • % for remainders (aka modulus)
  • () for grouping calculations






1.4: combining strings and numbers, a.k.a. concatenation

We can add strings to strings, or numbers to strings, to combine them.
This is called concatenation.









This only works with addition, a.k.a. +.
Using a different math operator results in an error.
