LESSON 2: <script>, functions, variables

In the previous lesson, we embedded JavaScript inside of HTML elements using the onclick attribute.
In this lesson, we’ll learn how to write JavaScript that spans multiple lines.

2.1: <script> and functions

The <style> element indicates CSS code.
Meanwhile, the <script> element indicates JavaScript code.

Inside of the <script> element, we can write things called functions.
Functions let us run multiple lines of JavaScript at once.
We can call a function just like we called alert() before.
Except, this time we refer to the function by the name we’ve given it.

function myFirstFunction() { alert("hi"); alert("what’s up!"); }


Functions can also pass arguments just like we did with alert() before.
We can then read those arguments inside of the function by naming them.


Unlike CSS classes, function names should NOT contain hyphens/dashes.
Instead, function names should just contain text.
One method for naming functions is using camel case.
This is when each word in the function name is capitalized, starting with the second word.
For example: myFunctionName

2.2: variables

Variables let us store data, refer to it later, and change it when we want.
There are three ways to create variables: var, let, and const.
For our purposes let’s just use var.

We create a variable by giving it a name and a value like this:
var myFirstVariable = "Hi there!";

And then we call a function with the variable name as its argument.
(Notice that we don’t put quotes around the variable’s name)
alert(myFirstVariable)




Note that variable names should also use camel case conventions (no hyphens/dashes!)

2.3: changing variables
